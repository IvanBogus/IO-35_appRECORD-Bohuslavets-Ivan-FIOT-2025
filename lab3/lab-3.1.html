Лабораторна робота №3. Верстка веб-сторінки засобами CSS і Flexbox

Тема: «Верстка веб-сторінки засобами CSS і Flexbox. Адаптивний інтерфейс для різних пристроїв».

Мета: набути навичок створення гнучких макетів веб-сторінок з використанням технології Flexbox, виконати верстку головної сторінки веб-застосунку готелю «Hotel Comfort», реалізувати семантичну структуру сторінки, навігацію, блок з пошуком номера, картки номерів та футер.

Посилання на виконані завдання лабораторної роботи:

Репозиторій власного веб-застосунку (GitHub) Власний веб-застосунок «Hotel Comfort» (жива сторінка) Репозиторій звітного HTML-документа (GitHub) Звітний HTML-документ (жива сторінка) Репозиторій самостійних робіт (GitHub) Самостійні роботи (живі сторінки)

Способи підключення стилів:

У сучасній розробці застосовують декілька способів підключення CSS: зовнішні таблиці стилів, внутрішні стилі в тегу <style> та вбудовані (inline) стилі через атрибут style.

У проєкті «Hotel Comfort» використано підхід із зовнішніми файлами стилів — окремий файл css/style.css підключається до сторінки за допомогою тега <link> у секції <head>. Додатково підключено нормалізацію стилів normalize.css та шрифти Google Fonts. 

<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <link rel="preconnect" href="https://fonts.googleapis.com" />
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
  <link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@400;500;600;700&display=swap" rel="stylesheet" />

  <link rel="stylesheet" href="css/style.css" />
</head>

Такий підхід робить код HTML-сторінки чистішим, дозволяє перевикористовувати стилі для кількох сторінок і спрощує підтримку проєкту.

Семантична структура сторінки:

Для побудови каркаса головної сторінки використано семантичні елементи <header>, <main>, <section>, <footer>, а також атрибути id для організації навігації по якорях. 

<body>
  <header class="header">...</header>

  <main>
    <section id="hero" class="hero">...</section>
    <section id="booking" class="booking">...</section>
    <section id="rooms" class="rooms">...</section>
    <section id="restaurant" class="stub-section">...</section>
    <section id="spa" class="stub-section">...</section>
    <section id="gallery" class="stub-section">...</section>
    <section id="contacts" class="contacts">...</section>
  </main>

  <footer class="footer">...</footer>
</body>

Така структура робить HTML-код зрозумілим як для розробника, так і для пошукових систем та assistive-технологій. Кожна секція відповідає логічному блоку: герой-секція, форма пошуку номерів, блок «Про нас», картки номерів, контакти тощо.

Результати роботи:

У результаті виконання першої частини лабораторної роботи №3 розроблено головну сторінку веб-застосунку «Hotel Comfort» зі структурованими блоками:

  • шапка сайту з логотипом, навігаційним меню та номером телефону;
  • герой-секція з основним заголовком, підзаголовком та кнопкою «Знайти номер»;
  • форма попереднього бронювання з полями дат і кількості гостей;
  • інформаційний блок «Більше, ніж просто готель» із текстом та міні-галереєю;
  • таблиця тарифів номерів і сітка карток номерів, реалізована засобами Flexbox/Grid;
  • блок контактів з адресою, телефоном, електронною поштою та формою зворотного зв’язку;
  • футер з копірайтом і посиланням на сторінку «Про проєкт».

Верстка виконана з використанням зовнішніх CSS-стилів, Flexbox та CSS-Grid для побудови гнучких сіток. Закладено основи адаптивності, які детальніше розглядаються у наступному підрозділі (3.2).
